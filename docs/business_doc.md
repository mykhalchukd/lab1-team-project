Бізнес-документ: Simple Product Catalog (SPC)

Назва та короткий опис:

Назва cистеми: Simple Product Catalog (SPC)
Simple Product Catalog (SPC) – це мінімалістичний веб-застосунок, розроблений як базова платформа для демонстрації, управління та відображення асортименту товарів. Система ідеально підходить для невеликих онлайн-магазинів, внутрішніх каталогів або як основа для більш складної системи управління запасами (Inventory Management System).

Основна мета SPC — забезпечити швидкий і надійний доступ до актуальних даних про товари. Система складається з простого клієнтського інтерфейсу, бекенду на Node.js, що надає REST API, та бази даних SQLite для зберігання даних про товари. У майбутньому планується розширення функціоналу для підтримки автентифікації та складніших операцій із даними.

Користувачі та ролі:

Система підтримує дві основні ролі з різними рівнями доступу:
Адміністратор - керує асортиментом товарів та налаштуванням системи. Основні права доступу: повний CRUD (Create, Read, Update, Delete) для товарів. Доступ до адміністративної частини API.
Публічний користувач - користувачі, які переглядають каталог без входу в систему. Основні права доступу: Перегляд списку товарів (Read). Пошук і фільтрація.

Основні сценарії використання

1. Перегляд каталогу - публічний користувач відкриває клієнтський інтерфейс і бачить повний, актуальний список усіх доступних товарів із цінами та основними характеристиками.
2. Управління товарною позицією (CRUD) - адміністратор входить у систему та додає нову товарну позицію, вказуючи назву, ціну, опис та початкову кількість на складі.
3. Пошук та фільтрація - користувач може використовувати рядок пошуку для швидкого знаходження товару за назвою або частиною опису, а також фільтрувати товари за ціною.
4. Редагування товарів - адміністратор вносить зміни до існуючих товарів (наприклад, оновлює ціну або залишок на складі).

Функціональні вимоги:

1. API для товарів - cистема повинна надавати REST API для отримання повного списку товарів (GET /items) та отримання одного товару за ID (GET /items/{id}).
2. CRUD-функціонал - cистема повинна підтримувати операції POST, PUT, та DELETE для керування товарними позиціями (доступно лише для Адміністратора).
3. Аутентифікація - cистема повинна мати базовий механізм аутентифікації (логін/пароль) для ролі Адміністратора.
4. Структура даних - кожен товар повинен містити такі обов'язкові поля: id, name, price, description та stock_quantity.
5. Клієнтський інтерфейс - клієнт повинен відображати товари у вигляді списку і мати інтерактивну кнопку для завантаження даних із бекенду.

Нефункціональні вимоги:

1. Швидкодія - час відгуку API для запитів GET /items не повинен перевищувати 500 мс при завантаженні до 1000 записів.
2. Безпека - усі операції зміни даних (POST/PUT/DELETE) повинні бути захищені токенною аутентифікацією (JWT або схожим механізмом).
3. Масштабованість - архітектура повинна дозволяти легку заміну SQLite на промислову СУБД (наприклад, PostgreSQL) у майбутньому.
4. Зручність(UX) - клієнтський інтерфейс має бути інтуїтивно зрозумілим і адаптивним (коректно відображатися на мобільних пристроях).
5. Технічне обслуговування - код повинен бути модульним, а всі залежності (Node.js) мають бути зафіксовані у package.json.

Обмеження:

Платежі - система не включатиме платіжний функціонал або інтеграцію з фінансовими шлюзами (PayPal, Visa/MasterCard).
Замовлення - SPC не буде обробляти кошики, замовлення або історію транзакцій.
Логістика - не буде механізмів відстеження посилок або розрахунку вартості доставки.
Мови - інтерфейс підтримуватиме лише одну мову (українську або англійську).

Пояснення архітектурних блоків діаграми:

Система Simple Product Catalog (SPC) побудована на трирівневій архітектурі, яка включає фронтенд, бекенд та базу даних.

| Блок | Призначення | Роль у Системі |
| :--- | :--- | :--- |
| **CLIENT (Browser)** | Клієнтський інтерфейс на **HTML, CSS та JavaScript**. | Відповідає за відображення каталогу товарів, взаємодію з користувачем (кнопки, пошук) та надсилання **API-запитів** до бекенду. |
| **BACKEND (Node.js/Express)** | Серверний додаток, написаний на Node.js з використанням фреймворку Express. | **Ядро системи**. Приймає HTTP-запити від Клієнта, обробляє бізнес-логіку, взаємодіє з базою даних і повертає відповіді у форматі JSON. |
| **DATABASE (SQLite/PG)** | Реляційна система управління базами даних (СУБД). На початковому етапі — **SQLite**, з можливістю переходу на **PostgreSQL (PG)**. | Місце для **постійного зберігання** даних (таблиці `items` та `users` для авторизації). |
| **Core Logic (Items)** | Модуль бекенду, який реалізує ендпоїнти `GET /items` та `GET /items/{id}`. | Основний функціонал для **читання даних** каталогу. |
| **Auth Service** (Майбутній) | Запланований модуль для управління **аутентифікацією** та **авторизацією** користувачів. | Відповідатиме за логіку входу (логін), видачу токенів (JWT) та перевірку прав доступу до адміністративних функцій. |
| **Admin CRUD** (Майбутній) | Запланований модуль для реалізації **операцій зміни даних** (Create, Update, Delete) для товарів. | Забезпечує API для Адміністратора для повного керування асортиментом. |
