# 0002. Layered Architecture (Modular Monolith)

## 1. Контекст
Проєкт використовує модульний моноліт для забезпечення чистоти коду, керованості та чіткого розділення відповідальності між шарами.

## 2. Рішення
Обираємо **Тришарову (Layered) Архітектуру**:
1.  **api/**: Рівень входу (Controllers, Routes).
2.  **service/**: Рівень бізнес-логіки (Use Cases, Validation).
3.  **domain/**: Рівень домену (Entities, Value Objects, Aggregates).
(Рівень інфраструктури (repository, db) буде додано пізніше, він підпорядковується service/).

## 3. Правила залежностей (The Dependency Rule)
Залежності можуть рухатися **тільки ззовні всередину**:
* `api/` може залежати від `service/` та `domain/`.
* `service/` може залежати лише від `domain/` та `infrastructure/`.
* `domain/` не може залежати ні від чого іншого (чиста бізнес-логіка).

## 4. Заборонені зв'язки
* **Заборона 1:** `api/` не може напряму звертатися до бази даних або `repository/`.
* **Заборона 2:** `domain/` не може імпортувати або знати про `api/` чи `service/`.
