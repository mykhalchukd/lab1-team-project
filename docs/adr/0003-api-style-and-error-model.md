# 0003. API Style and Error Model

## 1. Контекст
Необхідно встановити єдині правила для розробки RESTful API та повернення помилок для підвищення зручності інтеграції.

## 2. Рішення
1.  **API Style:** Використовуємо **RESTful API** (ресурсно-орієнтований, використовуючи HTTP-методи та статус-коди).
2.  **API Contract:** API має строго відповідати контракту, визначеному в **`openapi.yaml`**.
3.  **Error Model:** Для всіх помилкових відповідей (4xx, 5xx) використовується єдиний формат **`ErrorResponse`** (визначений у `openapi.yaml`) для уніфікованої обробки на стороні клієнта.
    * **Приклад:** 400 Bad Request, 404 Not Found, 409 Conflict завжди повертають тіло з цим об'єктом.
    * **Коди помилок (Error Codes):** Усі коди помилок (`code` у відповіді) мають бути унікальними та містити причину (наприклад, `TITLE_REQUIRED`, `ITEM_NOT_FOUND`).
